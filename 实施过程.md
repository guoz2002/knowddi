  ## 2026-04-18 环境兼容与基础适配

  ### 目标

  在现代运行环境下重新跑通原始 KnowDDI 代码，尽量只做框架兼容性修改，不改动原始训练逻辑。

  目标环境：

  - Python 3.12
  - PyTorch 2.3.0
  - CUDA 12.1
  - RTX 4090

  ### 已修改文件

  #### 1. `pytorch/utils/graph_utils.py`

  修改目的：

  - 兼容新版 DGL，替换旧版图构建方式。

  具体改动：

  - 去除旧版 `networkx -> dgl.from_networkx` 的构图路径。
  - 将图构建改为直接收集边的 `src/dst/type` 后，通过 `dgl.graph(...)` 构图。
  - 直接写入 `edata['type']`。

  结果：

  - 不改变图结构和关系类型语义，仅完成新版 DGL 接口适配。

  #### 2. `pytorch/data_processor/datasets.py`

  修改目的：

  - 兼容新版 DGL 的子图接口。

  具体改动：

  - 将 `if not ssp_graph:` 改为 `if ssp_graph is None:`。
  - 将旧版 `subgraph(...)` 接口替换为 `dgl.node_subgraph(...)`。
  - 使用 `store_ids=True` 显式保留原图索引。
  - 使用 `subgraph.edata[dgl.EID]` 回取原始边信息。
  - 在删除边的相关逻辑中增加异常保护。

  结果：

  - 子图提取流程适配到当前 DGL 版本，逻辑保持不变。

  #### 3. `pytorch/train.py`

  修改目的：

  - 避免原代码对 GPU 可见设备做硬编码。

  具体改动：

  - 删除固定写死的 `CUDA_VISIBLE_DEVICES='0,1,2,3'`。

  结果：

  - 训练逻辑不变，但单卡服务器上运行更稳定。

  #### 4. `pytorch/utils/initialization_utils.py`

  修改目的：

  - 提高现代 PyTorch 环境下模型加载稳定性。

  具体改动：

  - 在 `torch.load(...)` 中增加 `map_location=params.device`。

  结果：

  - 模型在不同设备上的加载更稳定，不改变模型内容。

  #### 5. `pytorch/manager/trainer.py`

  修改目的：

  - 兼容新版 PyTorch 的损失函数参数写法。

  具体改动：

  - 将 `nn.BCELoss(reduce=False)` 改为 `nn.BCELoss(reduction='none')`。

  结果：

  - 数学含义不变，仅完成新版本接口适配。

  #### 6. `requirements-modern.txt`

  修改目的：

  - 为现代环境补充独立依赖说明。

  具体改动：

  - 补充现代环境所需的依赖版本说明。

  结果：

  - 便于后续在新环境中重复部署。

  #### 3. 修复 Python 3.12 下 `random.sample` 与 `set` 的兼容问题

  定位文件：

  - `pytorch/utils/graph_utils.py`

  问题现象：

  - `random.sample(next_lvl, ...)` 在 Python 3.12 下对 `set` 不再直接兼容。

  修复方式：

  - 改为 `random.sample(sorted(next_lvl), ...)`。

  结果：

  - 不改变算法含义，仅修复 Python 版本兼容问题。

  #### 4. 修复 LMDB `map_size` 过大导致的内存映射报错

  定位文件：

  - `pytorch/data_processor/subgraph_extraction.py`

  问题现象：

  - 预处理阶段出现 `lmdb.MemoryError`，原因是初始映射空间设置过大。

  修复方式：

  - 降低初始 `map_size`。
  - 增加 `lmdb.MapFullError` 异常处理，在写入过程中动态扩容。

  结果：

  - 数据预处理流程能够继续推进，不再被初始映射空间问题阻塞。

  #### 5. 启动 DrugBank 预处理与训练前流程
