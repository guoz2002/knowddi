## 3. Python 3.12 与 random.sample 兼容性问题

### 报错

```python
TypeError: Population must be a sequence. For dicts or sets, use sorted(d).
```

### 原因

项目代码中对 `set` 直接调用了 `random.sample`，而 Python 3.12 不再接受这种写法。

### 位置

- `pytorch/utils/graph_utils.py`

### 修复

将：

```python
random.sample(next_lvl, max_nodes_per_hop)
```

改为：

```python
random.sample(sorted(next_lvl), max_nodes_per_hop)
```

### 结论

- 不改变原有采样逻辑
- 仅修复 Python 3.12 兼容性

## 4. LMDB 映射空间申请失败

### 报错

```python
lmdb.MemoryError: ... Cannot allocate memory
```

### 原因

原始代码在 `lmdb.open(..., map_size=...)` 时一次性申请较大的映射空间，容器环境无法完成该映射。

### 位置

- `pytorch/data_processor/subgraph_extraction.py`

### 修复思路

- 使用更保守的初始 `map_size`
- 如果后续写入触发 `lmdb.MapFullError`，则自动扩容

### 修复内容

初始值调整为：

```python
map_size = max(int(links_length * BYTES_PER_DATUM * 0.25), 512 * 1024 ** 2)
```

写入阶段增加自动扩容逻辑：

```python
while True:
    try:
        with env.begin(write=True, db=split_env) as txn:
            txn.put(str_id, serialize(datum))
        break
    except lmdb.MapFullError:
        env.set_mapsize(env.info()['map_size'] * 2)
```

### 结论

- 不改变数据处理逻辑
- 解决了预处理阶段因 LMDB 映射过大导致的启动失败

## 5. GPU 编号错误

### 报错

```python
dgl._ffi.base.DGLError: CUDA: invalid device ordinal
```

### 原因

训练默认参数中写死了 `gpu=3`，但服务器容器实际只暴露了 `gpu 0`。

### 检查结果

```text
count= 1
current= 0
0 NVIDIA GeForce RTX 4090
```

### 处理

训练时显式指定：

```bash
python train.py -e Drugbank --gpu 0
```

### 结论

- 问题来自运行参数，不是模型代码本身错误
- 后续训练需按实际服务器 GPU 编号手动指定

## 6. DataLoader worker 段错误

### 报错

```python
RuntimeError: DataLoader worker (...) is killed by signal: Segmentation fault.
```

### 原因判断

大概率与以下组合有关：

- `Python 3.12`
- `DGL + LMDB`
- 多进程 `DataLoader`
- 默认 `num_workers=32`

训练在多进程取数阶段不稳定，worker 进程随机崩溃。

### 处理建议

将 `num_workers` 显式调低：

```bash
python train.py -e Drugbank --gpu 0 --num_workers 4
```

如果仍不稳定，则进一步降到：

```bash
python train.py -e Drugbank --gpu 0 --num_workers 0
```

### 结论

- `num_workers` 影响训练稳定性与速度，不改变模型方法本身
- 为保证训练完成，优先选择更稳的 `num_workers=0`

## 7. checkpoint 是否可恢复训练

### 当前保存文件

服务器实验目录中已生成：

- `best_graph_classifier.pth`
- `log_train.txt`
- `params.json`

### 当前结论

项目当前只能加载模型权重，不能做“真正断点恢复训练”。

### 原因

现有代码只保存了模型本身，没有保存：

- 当前 epoch
- optimizer state
- scheduler state
- best metric
- early stopping 状态

### 可行方案

可以从当前最优模型权重继续训练：

```bash
python train.py -e Drugbank --gpu 0 --num_workers 0 --load_model
```

### 说明

这种方式属于“加载已有最优权重继续训练”，不是严格意义上的断点续训。
