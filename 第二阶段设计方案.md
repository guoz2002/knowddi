# KnowDDI 第二阶段设计方案

本文档用于补充当前材料中缺失的“方案设计”部分，重点回答第二阶段准备如何从现有 KnowDDI 基线出发，完成可实现、可验证、可消融的结构改造。本文档不再重复理论背景，而是直接面向现有代码给出工程化设计。

## 一、设计目标

第二阶段的核心目标不是重新搭建一套全新模型，而是在当前已经完成复现的 KnowDDI 基线之上，围绕知识子图学习过程引入更强的自适应结构建模能力，使模型能够在不同样本局部图上动态区分“应保留的信息”“应抑制的噪声”和“应补充的潜在关联”。

结合当前项目实际情况，第二阶段具体目标如下：

1. 保持现有数据预处理、子图抽取、训练入口和评估口径不变，保证改进前后实验具有可比性。
2. 将改动集中在 `pytorch/model/` 内部，优先改造图结构学习模块，避免对数据层和评估层造成额外干扰。
3. 将改进方案拆分为可独立验证的三个部分：自适应门控、结构去噪、结构补全。
4. 为后续第三阶段的代码实现和消融实验预留清晰的参数开关与实验命名方式。

## 二、现有基线结构梳理

根据当前代码，基线模型的主干流程如下：

1. `pytorch/train.py` 负责参数解析、数据集加载、训练器与评估器初始化。
2. `pytorch/data_processor/subgraph_extraction.py` 和 `pytorch/data_processor/datasets.py` 负责从原始 DDI 三元组与背景知识图谱中抽取局部子图，并组织成 DGL 图输入。
3. `pytorch/model/GraphSAGE.py` 负责在全局图上生成节点初始表示。
4. `pytorch/model/gsl_model.py` 负责构造批内全连接图，并通过图结构学习模块为边分配权重，再做多轮消息传播。
5. `pytorch/model/Classifier_model.py` 将子图级表示、头药物表示和尾药物表示拼接，最后完成 DDI 关系预测。
6. `pytorch/manager/trainer.py` 与 `pytorch/manager/evaluator.py` 负责训练、验证和测试。

其中，第二阶段最关键的可改进位置是 `pytorch/model/gsl_model.py`。当前实现已经具备“为完全连接图上的边学习权重”的能力，但总体仍属于统一的全局策略，主要体现在：

1. 所有边权学习共享同一套相似度函数，缺少对边类型和局部上下文差异的细粒度控制。
2. 当前“原始图边 + 潜在补全边”只通过 `lamda` 做固定融合，无法针对不同样本动态调节。
3. 当前稀疏化过程只依赖固定阈值 `threshold`，容易在稀疏样本和稠密样本之间产生同一策略、不同效果的问题。

因此，第二阶段的设计重点应放在“让边权生成从固定融合变为条件化决策”。

## 三、总体改造方案

第二阶段拟采用“自适应图结构解耦模块”替换当前单一的边权学习逻辑。整体流程如下：

原始局部子图
-> GraphSAGE 生成节点表示
-> 构造批内候选完全连接图
-> 自适应门控模块计算候选边得分
-> 去噪门控抑制低价值边
-> 补全门控增强高置信潜在边
-> 多层结构推理传播
-> 图级/头尾实体级表示融合
-> DDI 分类预测

为了便于实现和消融，整体方案拆成三层：

### 1. 基础层：边级自适应门控

对每条候选边，不再只输出一个简单相似度分数，而是结合以下信息生成门控值：

1. 源节点表示和目标节点表示。
2. 边关系类型嵌入。
3. 当前边是否来自原始子图的结构标记。
4. 节点在局部子图中的角色信息，例如是否为头实体、尾实体或普通邻居节点。

该层输出一个 `gate_score`，表示该边在当前样本上下文中应当被保留到什么程度。

### 2. 去噪层：原始边重加权

对于原始子图中已经存在的边，不再默认赋予固定增强权重，而是根据门控值决定：

1. 哪些边应当保留高权重。
2. 哪些边虽然存在于原图中，但对当前 DDI 判断贡献较低，应当被削弱。

去噪层的目标不是删除全部弱边，而是压低无效传播路径，减少噪声累积。

### 3. 补全层：潜在边选择性激活

对于完全连接图中新引入的潜在边，不再统一参与传播，而是仅允许高置信边通过门控进入消息传递过程。也就是说，补全不是“全部补全”，而是“按样本条件补全”。

补全层的目标是在局部子图过于稀疏时，为模型提供额外结构连接，提升隐含关系表达能力。

## 四、模块级设计

### 1. `pytorch/model/gsl_model.py`

该文件是第二阶段改造的核心位置，建议按以下方式扩展。

#### 设计一：保留当前 `graph_structure_learner` 主体框架

原因：

1. 当前代码已经完成候选图构造、边类型嵌入、edge softmax 和稀疏化流程。
2. 直接在现有类中扩展，改动最集中，便于与基线对照。
3. 不需要重写整个训练链路，风险较低。

#### 设计二：新增边门控特征

当前 `compute_similarity()` 主要依据节点差异和关系嵌入。第二阶段建议新增以下输入：

1. `is_original_edge`：标识该边是否来自原始子图。
2. `src_role` 与 `dst_role`：标识节点是否为头实体、尾实体或普通节点。
3. 可选的局部结构统计特征，例如节点度或局部标签。

实现上可以将这些信息拼接到当前 MLP 输入中，形成新的边级门控网络。

#### 设计三：将单一权重拆为两路分数

当前只有一个 `weight`。第二阶段建议拆分为：

1. `denoise_score`：控制原始结构边的保留强度。
2. `complete_score`：控制潜在补全边的激活强度。

最终边权可设计为：

`final_weight = alpha * denoise_score + beta * complete_score`

其中 `alpha`、`beta` 可为超参数，也可进一步由门控网络动态输出。

这样做的好处是后续可以方便支持：

1. 仅去噪实验。
2. 仅补全实验。
3. 去噪 + 补全联合实验。

#### 设计四：保留兼容模式

为保证与基线结果直接对照，建议在该文件中保留“原始 GSL 模式”和“改进 AGSL 模式”两套路径，通过参数开关控制。这样后续不必维护两份模型代码。

### 2. `pytorch/model/Classifier_model.py`

该文件不需要大改，但需要承担改进模块的总入口职责。

建议调整如下：

1. 保持 `GraphSAGE + gsl_model + 分类头` 的总结构不变。
2. 通过参数决定调用原始结构学习模式还是改进后的自适应模式。
3. 如有需要，可额外保存中间门控统计量，供实验分析或可解释性展示使用。

换句话说，`Classifier_model.py` 主要作为集成层，不是算法创新主战场。

### 3. `pytorch/train.py`

第二阶段方案要真正可执行，必须在训练入口增加对应实验参数。建议新增如下参数：

1. `--gsl_mode`
   可选值：`baseline`、`adaptive`
   用于切换原始结构学习与改进结构学习。

2. `--use_denoise`
   可选值：`0/1`
   控制是否开启去噪模块。

3. `--use_completion`
   可选值：`0/1`
   控制是否开启补全模块。

4. `--gate_hidden_dim`
   用于控制门控网络隐藏层维度。

5. `--gate_dropout`
   用于控制门控网络 dropout。

6. `--completion_topk` 或 `--completion_threshold`
   用于控制潜在边保留数量或激活阈值。

通过这些参数，后续实验可直接形成标准命令组合，便于复现实验记录。

### 4. `pytorch/manager/trainer.py`

训练主逻辑可以基本保持不变，但建议补充两类记录：

1. 每轮训练中平均保留边比例。
2. 每轮训练中补全边激活比例。

这样做的目的不是影响训练，而是验证“模块到底有没有工作”，避免后续只看到最终指标，却无法解释结构变化是否真实发生。

### 5. `pytorch/manager/evaluator.py`

评估指标本身不需要改动，但建议在测试阶段附加记录：

1. 改进模型相较基线的核心指标变化。
2. 在 DrugBank 与 BioSNAP 上不同模块组合的性能差异。

如果后续要写论文，这部分可直接转化为消融实验表格。

## 五、方案实现路径

为了控制风险，第二阶段不建议一次性把所有改动同时接入，而应分三步推进。

### 第一步：自适应门控替换固定边权学习

目标：

1. 保持原始“构造完全连接图 -> 学习边权 -> 消息传播”的主流程不变。
2. 先将边权生成方式从单一相似度学习替换为带上下文特征的门控网络。

预期结果：

1. 模型能够动态地区分不同边的重要性。
2. 至少在验证集上表现不弱于当前基线。

### 第二步：加入去噪分支

目标：

1. 针对原始图中已有边进行重加权。
2. 验证仅做噪声抑制时，是否已经能带来收益。

预期结果：

1. 稠密子图场景下，去噪分支应更容易体现作用。
2. 即使最终指标提升有限，也应在边保留比例上体现更明确的选择性。

### 第三步：加入补全分支

目标：

1. 在保留去噪分支的基础上，再对潜在边做高置信补全。
2. 验证补全是否能改善稀疏样本建模。

预期结果：

1. 稀疏结构样本上效果更明显。
2. 如果补全边过多造成噪声增加，应通过阈值或 top-k 进一步约束。

## 六、消融实验设计

为了证明改进方案有效，至少应设置以下四组实验：

1. 基线模型
   即当前复现的原始 KnowDDI。

2. 基线 + 自适应门控
   验证“动态边权学习”本身是否有效。

3. 基线 + 自适应门控 + 去噪
   验证去噪模块的边际贡献。

4. 基线 + 自适应门控 + 去噪 + 补全
   验证完整方案是否进一步提升。

如时间允许，还可增加：

5. 基线 + 自适应门控 + 补全
   用于单独观察补全作用。

统一评价指标如下：

1. DrugBank：Macro-F1、ACC、Cohen's kappa。
2. BioSNAP：AUROC、AUPRC、AP@50。

统一比较原则如下：

1. 使用相同数据划分。
2. 使用相同训练轮数和早停策略。
3. 使用相同随机种子。
4. 优先报告验证集最优模型对应的测试结果。

## 七、预期成果形式

第二阶段完成后，应至少形成以下可交付内容：

1. 一份清晰的结构设计文档。
2. 一版可运行的改进模型代码。
3. 一组初步消融实验结果。
4. 一份记录各参数组合与实验结果的对照表。

这样第三阶段就不再是“边做边想”，而是按照既定模块逐步实现和验证。

## 八、与现有代码的对应关系

为避免方案空泛，现将第二阶段设计与当前代码文件对应如下：

1. `pytorch/model/gsl_model.py`
   第二阶段核心改造文件，负责门控、去噪、补全三部分逻辑。

2. `pytorch/model/Classifier_model.py`
   负责集成改进后的结构学习模块，并向分类头输出最终表示。

3. `pytorch/train.py`
   负责新增实验参数与模式切换入口。

4. `pytorch/manager/trainer.py`
   负责训练阶段统计信息记录。

5. `pytorch/manager/evaluator.py`
   负责实验结果汇总和对比输出。

因此，第二阶段的设计方案已经可以明确概括为：

在不改变数据层和评估层基本框架的前提下，以 `gsl_model.py` 为核心，对原始 KnowDDI 的图结构学习过程进行自适应门控改造，并通过去噪与补全两条分支构成可消融、可验证的结构增强方案。

## 九、下一步实施建议

建议后续代码实现严格按以下顺序推进：

1. 先在 `pytorch/train.py` 中补齐参数入口。
2. 再在 `pytorch/model/gsl_model.py` 中实现自适应门控主干。
3. 先跑“基线 vs 自适应门控”最小对照实验。
4. 确认改进方向合理后，再继续接入去噪和补全分支。
5. 最后统一整理实验表格与文档表述。

按这个顺序推进，风险最小，也最方便在中期材料和论文中交代“从设计到实现再到验证”的完整链路。
