# KnowDDI 机制解耦研究实施计划

  本文档基于当前课题研究目标制定，旨在为基于 KnowDDI 的图神经网络 DDI 预测研究提供清晰的阶段性实施路径，并统一后续实验指标与论文写作口径。本文工作的重点不在于单纯追求模型性能最优，而在于围绕知识子图学习中的结构演化机制开展定量解耦与机理分析。

  ## 一、总体研究目标

  本课题以 KnowDDI 为基础模型，围绕药物-药物相互作用预测任务展开研究，重点从以下几个方面推进：

  1. 完成 KnowDDI 在 DrugBank 与 BioSNAP 两个基准数据集上的有效复现。
  2. 在复现基础上研究知识子图学习中的自适应去噪与补全机制，并对两类结构操作进行定量解耦分析。
  3. 探索适用于边缘部署场景的轻量化结构改造方案。
  4. 通过消融实验和效率评估分析不同结构机制的边际贡献、作用边界及其计算代价。

  ## 二、指标口径统一说明

  为确保与原论文保持一致，后续实验统一采用如下评价指标：

  ### 1. DrugBank 数据集
  采用以下三项核心指标：
  - Macro-F1
  - ACC
  - Cohen's κ

  ### 2. BioSNAP 数据集
  采用以下三项核心指标：
  - AUROC
  - AUPRC
  - AP@50

  说明：
  - DrugBank 与 BioSNAP 的任务形式不同，因此不采用完全相同的评价指标。
  - 后续论文写作、实验对照表和消融分析均按照上述口径统一开展。

  ## 三、阶段划分与实施内容

  ### 第一阶段：环境搭建与基准复现
  目标：完成代码运行、数据准备与原始模型复现。

  主要任务：
  1. 配置本地与服务器实验环境，完成 Python、PyTorch、PyTorch Geometric 及相关依赖安装。
  2. 导入并整理 DrugBank、BioSNAP 与 Hetionet 数据。
  3. 跑通 KnowDDI 原始训练与评估流程。
  4. 记录核心评价指标基准数值：
     - DrugBank：Macro-F1、ACC、Cohen's κ
     - BioSNAP：AUROC、AUPRC、AP@50
  5. 梳理模型保存逻辑、评估逻辑和解释性子图输出格式。

  阶段性目标：
  - 成功复现原始模型
  - 获得与原论文接近的基准结果
  - 形成后续机制分析实验的对比基线

  ### 第二阶段：模型机制分析与解耦方案设计
  目标：明确原始模型中的结构学习机理与局限，并完成面向机制分析的解耦方案设计。

  主要任务：
  1. 分析原模型中知识子图学习、去噪与补全模块的工作机制。
  2. 研究固定全局策略在不同样本稀疏度下的局限性。
  3. 设计基于注意力门控的自适应图结构解耦机制。
  4. 明确解耦模块的输入、输出及与原训练流程的衔接方式。

  阶段性目标：
  - 完成机制解耦方案理论设计
  - 明确核心研究假设
  - 为代码实现与机制分析实验做好准备

  #### 第二阶段理论任务完成说明

  为完成第二阶段的理论分析任务，当前已围绕 KnowDDI 原始模型的结构机理、局限性来源以及可行解耦方向完成系统梳理，形成如下阶段性结论。

  1. 已完成原模型核心机制分析。通过对编码器、知识子图学习模块与最终分类流程的拆解，明确了 KnowDDI 的整体工作链路：首先利用药物对的局部子图抽取结构化邻域信息，然后通过图神经网络完成节点表示传播，最后结合关系层面的交互建模完成 DDI 预测。该分析说明，预测结果不仅取决于节点表示质量，还取决于子图中保留了多少有效关系、抑制了多少噪声连接。
  2. 已完成原模型局限性归纳。理论分析表明，原始 KnowDDI 在不同样本和不同关系类型下采用相对统一的结构处理方式，容易出现两类问题：一是对子图内冗余边和弱相关边缺乏针对性的抑制，导致噪声传播；二是对潜在有用但在局部图中表现不充分的关联缺乏补偿机制，导致关键信息利用不足。这说明固定全局策略在样本稀疏度和关系复杂度变化时适应性有限。
  3. 已明确研究问题的理论切入点。基于上述分析，后续工作不再直接追求“更深的传播层数”或“更大的隐藏维度”，而是转向考察知识子图结构本身的可适配性，即让模型能够根据样本内容动态决定哪些连接应被强化、哪些连接应被削弱，从而分析结构噪声与结构缺失对最终预测的影响。
  4. 已形成自适应图结构解耦的理论方案。该方案的核心思想是在原有知识子图学习框架上引入注意力门控机制，根据节点表示、关系语义以及局部上下文信息，为边分配动态权重，并将其划分为“保留信息”“抑制噪声”和“待补全关联”三类结构信号。这样可以在不破坏原模型总体框架的前提下，使子图学习过程从静态筛选转向条件化建模。
  5. 已明确去噪与补全模块的功能定位。去噪模块负责抑制对预测贡献较低甚至带来误导的连接，目标是减少无效信息传播；补全模块负责在原始局部结构较稀疏时补入高置信度的潜在关联，目标是增强模型对隐含关系的表达能力。两者在理论上并非互斥，而是分别对应“减少错误信息输入”和“增加有效信息覆盖”两个方向。
  6. 已完成解耦模块与原训练流程的接口设计。理论方案中保持原数据预处理、子图抽取、训练入口和评估流程不变，改动主要集中在模型前向传播中的结构权重学习部分。这样可以保证后续实验中基准模型与解耦模型共享同一数据划分、同一训练策略和同一评价口径，便于公平比较不同结构机制的作用差异。
  7. 已完成第二阶段核心假设定义。后续实验将重点检验以下假设：在知识子图学习中引入自适应门控后，模型能够更有效地区分关键关系与噪声关系；在此基础上叠加适度的结构补全，可进一步影响模型对稀疏样本和复杂关系模式的建模方式；若该假设成立，则去噪与补全模块应在不同数据集和不同评价指标上表现出可观测、可解释的差异化边际贡献。

  第二阶段的理论任务至此已基本完成，已经具备进入第三阶段代码接入与消融实验的条件。后续实现工作将严格围绕上述理论分析展开，优先检验自适应解耦、去噪与补全三个部分的实际贡献。

  ### 第三阶段：机制解耦实现与消融验证
  目标：完成解耦模块接入，并定量分析去噪与补全两类结构操作对模型行为和预测结果的真实边际贡献。

  主要任务：
  1. 将自适应门控机制集成到原始 KnowDDI 模型中。
  2. 分别设计“仅去噪”“仅补全”“去噪+补全”三类对照实验。
  3. 在 DrugBank 与 BioSNAP 上开展消融实验。
  4. 定量分析去噪与补全对核心指标、不同数据集及不同任务设定的边际影响。

  里程碑：
  - 定量输出去噪与补全模块对核心指标的真实边际贡献
  - 识别去噪与补全在不同任务和不同指标上的差异化作用
  - 明确相关机制的适用边界，而非仅以是否全面优于基线作为结论依据

  ### 第四阶段：轻量化重构与部署评估
  目标：在保证核心评价结果基本可接受的前提下，降低模型计算开销，并评估轻量化后的表征稳定性与边缘部署可行性。

  主要任务：
  1. 探索子图采样稀疏化策略，减少图传播过程中的冗余计算。
  2. 尝试特征维度压缩与网络层数精简，构建轻量化模型变体。
  3. 记录参数量、显存占用、单次推理延迟等效率指标，建立精度-效率对照关系。
  4. 分析轻量化后核心评价指标的变化是否处于可接受范围。
  5. 检查压缩后模型在小样本与稀疏场景下的结果稳定性，评估是否出现明显表征退化或语义漂移。

  重点评估内容：
  - DrugBank 与 BioSNAP 上核心指标变化
  - 参数量变化
  - 显存占用变化
  - 推理时延变化
  - 轻量化后结果稳定性与语义保真情况

  里程碑：
  - 构建“精度-效率”对照结果
  - 明确轻量化方案在不同数据集上的可接受精度损失范围
  - 说明压缩后模型是否仍具备基本稳定的表征能力与部署可行性
  - 为后续边缘部署论证提供依据

  ### 第五阶段：结果整理与论文写作
  目标：完成实验总结、图表整理和论文主体撰写。

  主要任务：
  1. 汇总基准复现结果、机制解耦实验结果、消融实验结果和轻量化实验结果。
  2. 形成统一的实验对照表与图表材料。
  3. 整理原论文结果与本研究结果的对比关系。
  4. 撰写论文中的实验设计、结果分析和结论部分。
  5. 预留正式参考文献引用位置，后续统一替换为标准格式。

  阶段性目标：
  - 完成论文主体实验章节
  - 完成结果对照表
  - 完成答辩材料基础版本
