# 🚀 KnowDDI 改进项目实施计划

本文档基于开题报告生成，旨在为基于 KnowDDI 的图神经网络 DDI 预测模型的改进工作提供清晰的阶段性实施路径。

## 📍 总体目标
1. **定量解耦**：引入自适应注意力门控，解耦图结构学习中的“去噪”与“补全”模块并定量分析。
2. **轻量化重构**：针对边缘计算场景，通过拓扑、参数协同压缩，降低推理延迟和显存占用。

---

## 📅 第一阶段：环境搭建与 KnowDDI 基准复现
**目标**：确立原始模型的多指标性能基准，了解现有代码结构与数据流转。

- [ ] **任务 1.1：开发环境配置**
  - 搭建基于 Python, PyTorch 及 PyTorch Geometric (PyG) 的深度学习运行环境。
- [ ] **任务 1.2：数据集准备**
  - 导入并预处理 DrugBank 和 BioSNAP 双路数据集及外部生物医学知识图谱（Hetionet）。
- [ ] **任务 1.3：基准复现与指标记录**
  - 运行原始 KnowDDI 训练代码（`pytorch/train.py`）。
  - 记录核心性能指标初始基准数值（F1-score, ACC, AUROC）。
- [ ] **任务 1.4：子图分析**
  - 深入解析原始模型生成的解释性子图的输出格式，为后续“语义漂移”评估建立数据基础。

---

## 🧠 第二阶段：基于注意力门控的图结构学习自适应解耦
**目标**：打破原模型的全局固定策略，实现样本特异性的动态结构演化，并完成定量分析。

- [ ] **任务 2.1：门控机制设计与开发**
  - 在原有的 Resembling（补全）和 Pruning（去噪）模块前，设计并嵌入轻量级注意力门控单元。
  - 根据输入药物节点的连接密度，自动分配增强（加边）或削减（减边）的动态权重。
- [ ] **任务 2.2：模块代码整合**
  - 遵循项目代码规范，在 `pytorch/model/gsl_model.py` 等文件中完成逻辑替换，确保向后兼容。
- [ ] **任务 2.3：消融实验与扰动敏感度测试**
  - 构建受控变体模型，通过改变图结构的稀疏扰动比例进行系统消融实验。
  - **里程碑**：定量输出自适应解耦模块对小样本预测精度（F1/AUROC）的真实边际贡献。

---

## ⚡ 第三阶段：面向边缘端场景的模型轻量化重构
**目标**：解决 GNN 参数冗余与高延时痛点，实现模型在资源受限场景下的 Pareto 最优部署。

- [ ] **任务 3.1：拓扑压缩（稀疏化子图采样）**
  - 弃用全邻域传播，开发基于“节点中心性”的自适应随机采样算法，保留高贡献路径，降低 Laplacian 计算维度。
- [ ] **任务 3.2：参数与算子压缩**
  - 压缩高维药物特征向量使其通过线性瓶颈层（维度 $d$ 优化）。
  - 缩减 GNN 堆叠迭代层数（深度 $T$ 优化）。
  - （可选）引入知识蒸馏思想，用浅层网络保留深层拓扑语义。
- [ ] **任务 3.3：计算性能 Profile（效率评估）**
  - 借助 PyTorch Profiler 模拟受限硬件环境运行。
  - **里程碑**：统计记录并对比改进前后的单次推理延时 (Latency)、显存峰值 (Peak VRAM) 和精度损失，描绘 "精度-效率" Pareto 曲线。

---

## 📊 第四阶段：机理验证、分析与总结
**目标**：全面汇总数据，验证改进逻辑的临床应用有效性与生物学透明度。

- [ ] **任务 4.1：可解释性语义保真度检验**
  - 选取具有明确药理机制的复杂用药配对（特别在小样本极端稀疏场景）。
  - 对比原模型与轻量化改进模型生成的局部子图，验证核心药理逻辑未发生“语义漂移”。
- [ ] **任务 4.2：可视化与数据汇总**
  - 整理多维实验数据，制作对比图表。
  - 从机理层面清晰阐述图结构学习在解决小样本 DDI 问题上的核心优势。
- [ ] **任务 4.3：论文/报告撰写准备**
  - 整理上述输出的客观实验证据，形成可用于论文写作的材料，重点突出创新点：解耦定量分析与边缘端轻量化重构。
